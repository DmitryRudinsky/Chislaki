# Решение системы нелинейных уравнений

## Описание задачи

Решить систему нелинейных уравнений методом Ньютона, методом простой итерации и методом Зейделя с точностью ε = 10⁻⁴ (в UI) и максимумом 200 итераций:

```
x₁² + x₂² - 2cos(x₁) = 3
e^(x₁²-1) - x₂ = 3
```

## Структура проекта

- `index.html` - Веб-интерфейс приложения
- `numerical_methods.js` - Реализация численных методов
- `graph.js` - Визуализация и анализ графиков
- `main.js` - Electron приложение (опционально)
- `package.json` - Настройки проекта

## Запуск

## Функциональность

### 1. Графическое определение корней
- Автоматическое построение графиков эквивалентных функций:
  - x₂ = ±√(3 - x₁² + 2cos(x₁))
  - x₂ = e^(x₁²-1) - 3
- Поиск точек пересечения графиков
- Автоматическое заполнение начальных приближений

### 2. Численные методы

#### Метод Ньютона
- Использует матрицу Якоби
- Квадратичная сходимость (локально)
- Критерий остановки: |Δx| < ε и ||F(x)|| < ε

#### Метод простой итерации
- Итератор с релаксацией по первой компоненте: φ₁(x) = x₁ + α·F₁(x), φ₂(x) = e^(x₁²-1) - 3
- Альтернатива с релаксацией (ω = 0.2)
- Проверка локальной сходимости: оценка sup||Φ'(x)|| < 1 в окрестности старта
- Автоматическое переключение на релаксационный метод при расхождении

#### Метод Зейделя
- Модификация метода простой итерации с последовательным обновлением
- Оценка локальной сходимости: sup||Φ'(x)|| < 1 в окрестности старта
- Обычно сходится быстрее простой итерации

### 3. Анализ результатов
- Сравнительная таблица всех методов
- Количество итераций для каждого метода
- Проверка условий сходимости
- Финальная погрешность

## Теоретические основы

### Система уравнений
Исходная система преобразуется к эквивалентным функциям для графического анализа и численного решения.

### Условия сходимости
- **Ньютон**: Матрица Якоби должна быть невырожденной
- **Простая итерация**: Норма матрицы производных < 1
- **Зейдель**: Аналогично простой итерации

### Точность
По умолчанию UI: ε = 10⁻⁴, максимум 200 итераций (настраивается в интерфейсе). В вспомогательных скриптах для поиска корней может использоваться более строгая ε = 10⁻⁶.

## Особенности реализации

1. **Автоматический поиск корней** через анализ пересечений графиков
2. **Интерактивная визуализация** с возможностью масштабирования
3. **Проверка условий сходимости** для каждого метода
4. **Подробная трассировка** итерационного процесса
5. **Сравнительный анализ** эффективности методов
6. **Улучшенные итерационные методы** с релаксацией для лучшей сходимости
7. **Автоматическое переключение** между вариантами методов при расхождении

## Исправления математических методов

### Проблемы в исходной версии:
- Недостаточный критерий остановки Ньютона (только |Δx|)
- Точечная проверка ||Φ'|| в одной точке
- Несогласованность параметров ε, maxIter, ω

### Внесенные исправления:
1. **Метод Ньютона**:
   - Добавлен двойной критерий остановки: |Δx| < ε и ||F(x)|| < ε
   - Исправлен подсчет числа итераций

2. **Простая итерация / Зейдель**:
   - Добавлена оценка sup-нормы Якоби Φ на прямоугольнике вокруг старта
   - Явное использование релаксации; альтернативная схема с ω = 0.2

3. **Согласование параметров**:
   - UI: ε = 10⁻⁴, maxIter = 200; релаксация ω = 0.2, α = -0.05
   - Документация и интерфейс приведены в соответствие
